channel

值尺寸

1 word

字面量

  • 双向通道: chan EleType
  • 只读通道: <-chan EleType
  • 只写通道: chan<- EleType

零值

nil

内部结构

一个通道有 3 个队列和一个互斥锁维护数据同步

  • 数据缓冲队列: BufferSize 长度的循环队列
  • 接收协程队列: 阻塞的等待从通道接收数据的协程
  • 发送协程队列: 阻塞的等待向通道发送数据的协程

操作

  • 关闭 close(channel)
  • 发送 channel<-
  • 接收 <-channel: 返回值有两个，第二个可选返回值是 bool，true 表示正常接到从数据缓冲区或未阻塞的发送协程发来的数据，false 表示从已关闭的非零通道接收到 EleType 零值，但是这个值不能用来判断 channel 是否关闭，因为一个已关闭的非零通道在数据缓冲区有数据时也返回 true。
  • len(channel) 和 cap(channel)

    尽管上述通道的操作是同步，但是通道的赋值和从通道接收出来值进行赋值却是跟其它类型的一样是未同步的
    a := <-ch // 通道接收值并赋值给其它值
    ch = AnotherCh // 通道赋值

  • for v := range ch: 不断尝试从通道中获取数据直到通道关闭并且数据缓冲队列数据清空

    等价于

  • select case
  • 通道元素值的传递都是值复制
  • 数据接收和发送操作是简单语句

close panic panic 1
send block panic 5 2
receive block 3 4
  1. 关闭成功
  • 接收协程队列不为空 => 数据缓冲区为空 => 弹出等待的协程，接收到 EleType 的零值，不阻塞
  • 发送协程队列不为空 => 数据缓冲队列已满 => 弹出等待的协程，等待的协程将 panic
  1. 阻塞或者发送成功
  • 缓冲通道
    • 接收协程队列不空 => 数据缓冲队列为空 => 弹出一个接收协程接收发送协程的值，发送协程不阻塞
    • 接收协程队列空 => 数据缓冲队列不满 => 发送协程的值存储到数据缓冲队列，发送协程不阻塞
    • 接收协程队列空 => 数据缓冲队列已满 => 发送协程加入到发送协程队列，发送协程阻塞
  • 非缓冲通道
    • 接收协程队列为空 => 加入发送协程队列，阻塞
    • 接收协程队列不空 => 弹出一个接收协程接收数据，不阻塞
  1. 永不阻塞
  • 数据缓冲队列尚有数据 => 接收协程接收到缓冲的值
  • 数据缓冲队列没有数据 => 接收到 EleType 类型的零值
  1. 阻塞或者接收成功
  • 缓冲通道
    • 发送协程队列不为空 => 缓冲队列已满 => 从缓冲队列接收数据，不阻塞
    • 发送协程队列空且数据缓冲队列未满 => 从数据缓冲队列接收数据，不阻塞
    • 发送协程队列空且数据缓冲队列空 => 接收协程加入接收协程队列，阻塞
  • 非缓冲通道
    • 发送协程队列不为空 => 弹出发送协程，接收协程接收发送协程的数据，不阻塞
    • 发送协程队列空 => 加入接收协程队列，阻塞
  1. panic 的原因: 已关闭的通道，其发送协程队列不接受任何协程加入
